# 期末考试自测工具 — 设计文档

**日期**: 2026-06-20
**状态**: 设计中
**类型**: Web 应用（纯前端）

---

## 1. 概述

### 1.1 目标

构建一个纯前端 Web 应用，用户可以上传题库文件（PDF/DOCX/PPTX），系统自动解析出选择题、多选题、填空题、简答题，支持逐题作答，提交后自动评分并展示总分和逐题对错情况。

### 1.2 核心原则

- **纯前端**：所有处理在浏览器完成，文件不上传服务器，隐私安全
- **即开即用**：无需注册、无需后端、无需数据库
- **自动解析**：基于正则匹配自动识别题目结构，无需手动标注
- **本地存储**：答题进度和记录保存在 LocalStorage

## 3. 架构

### 3.1 系统流程

```
上传文件 → 文本提取 → 题目解析 → 逐题作答 → 提交评分 → 成绩展示
```

### 3.2 数据模型

```typescript
interface Question {
  id: string;                    // "ch1-q3"
  type: 'single' | 'multiple' | 'true-false' | 'fill-blank' | 'short-answer';
  stem: string;                  // 题干
  options: string[];             // 选项列表（简答题为空数组）
  answer: string;                // 标准答案（单选: "A", 多选: "ABD", 填空: "文本", 简答: "完整答案"）
  explanation: string;           // 解析（可选，从文件中提取）
  chapter: string;               // 所属章节
  score: number;                 // 题目分值（默认每题 5 分）
}

interface QuizSession {
  questions: Question[];         // 题目列表
  userAnswers: Record<string, string>;  // 用户答案 { questionId: answer }
  currentIndex: number;          // 当前题目索引
  startTime: number;             // 开始时间
  submitted: boolean;            // 是否已提交
}
```

## 4. 文件解析

### 4.1 文本提取

| 文件类型 | 解析方式 |
|---|---|
| PDF | pdf.js 逐页提取文本内容，保留换行 |
| DOCX | mammoth.js 转 HTML，再提取纯文本，保留段落结构 |
| PPTX | JSZip 解压，解析 slide XML，提取 `<a:t>` 文本节点，按 slide 顺序拼接 |

### 4.2 题目解析

基于正则表达式分段识别：

**分割题目**：`/^\s*(\d+)\.\s*\[/gm` — 匹配题号边界

**提取选项**：`/^([A-D])\.\s*(.+)/m` — 匹配选项行

**提取答案**：`/答案[:：]\s*(.+)/` — 匹配标准答案行

**题型推断规则**：

| 条件 | 题型 |
|---|---|
| 答案为单个字母且选项 ≥ 3 个 | 单选题 |
| 答案包含多个字母 (如 AB, ACD) | 多选题 |
| 答案为单个字母且只有 2 个选项 | 判断题 |
| 题干含 `( )` 或 `（）`，且答案不为字母 | 填空题 |
| 题干含 `( )` 或 `（）`，且答案为字母（选择填空） | 填空题 |
| 答案为空或答案长度 > 5 个字符，且无 A-D 选项 | 简答题 |

### 4.3 容错处理

- 解析后展示题目统计（各题型数量），但不设确认页
- 用户可在答题过程中通过答题卡发现解析问题
- 预留手动编辑题目类型的入口（答题卡右键或长按）

## 5. 答题流程

### 5.1 页面结构（桌面端）

```
┌─────────────┬──────────────────────────┐
│  答题卡      │                          │
│  (280px)    │   题目内容                │
│             │                          │
│  ┌──┬──┐   │   5. 题干文字...          │
│  │1 │2 │   │                          │
│  ├──┼──┤   │   ○ A. 选项一             │
│  │3 │4 │   │   ● B. 选项二             │
│  ├──┼──┤   │   ○ C. 选项三             │
│  │5*│6 │   │   ○ D. 选项四             │
│  └──┴──┘   │                          │
│             │   ← 上一题   下一题 →     │
│  ■ 已答     │                          │
│  □ 未答     │   进度 ████░░░░ 6%       │
│  ▣ 当前     │                          │
└─────────────┴──────────────────────────┘
```

### 5.2 答题卡

- **桌面端**：固定在左侧边栏，始终可见
- **移动端**：底部按钮「📋 答题卡」→ 弹出底部抽屉面板
- **颜色规则**：
  - 深绿色 `#10b981` = 已作答
  - 浅灰色 `#e2e8f0` = 未作答
  - 蓝色边框 = 当前题目
- **交互**：点击题号跳转到该题，已填答案保留
- **统计**：顶部显示「已答数/总数」

### 5.3 答题交互

- 选择/判断题：点击选项即选中，自动跳下一题（可配置开关）
- 多选题：点击切换选项选中状态，有「确认」按钮再跳下一题
- 填空/简答：文本输入框，需手动点「下一题」或按 Ctrl+Enter
- 上一题可回看修改
- 最后一题显示「提交」按钮

### 5.4 末题提交

- 最后一题按钮变为「提交答卷」
- 点击后弹出确认框，显示未答题数：「还有 N 题未作答，确定提交吗？」
- 确认后进入成绩页

## 6. 评分逻辑

### 6.1 评分规则

| 题型 | 评分方式 | 满分条件 | 零分条件 | 部分得分 |
|---|---|---|---|---|
| 单选题 | 精确匹配 | 选择 = 标准答案 | 选错 | — |
| 多选题 | 精确匹配 | 完全选对 | 有任何一个选错 | 漏选得一半分 |
| 判断题 | 精确匹配 | 判断正确 | 判断错误 | — |
| 填空题 | 精确匹配 | 答案完全一致（去首尾空格） | 任何不一致 | — |
| 简答题 | 关键词 + 相似度 | 关键词覆盖率 ≥ 80% | 覆盖率 < 30% | 50%-80% → 70%分；30%-50% → 40%分 |

### 6.2 简答题评分算法

1. **关键词提取**：从标准答案中提取长度 ≥ 2 的关键词，去掉停用词（的、了、是、和、或、等）
2. **关键词覆盖率**：用户答案中包含的关键词数 ÷ 标准答案关键词总数
3. **文本相似度**：Jaccard 系数 + 最长公共子序列 (LCS) 组合得分
4. **最终得分**：覆盖率 × 0.6 + 相似度 × 0.4，然后映射到评分档位

### 6.3 分值

- 默认每题 5 分
- 总分 = 各题得分之和
- 百分比显示：得分 ÷ 满分 × 100%

## 7. 成绩页

### 7.1 总览

- 大字显示总分（如「82.5 分」）
- 正确率百分比
- 统计标签：✓ 正确 N · ✗ 错误 N · △ 部分 N

### 7.2 逐题详情

- 滚动列表，每题一行
- 显示：题号、题干摘要、你的答案 vs 标准答案、得分
- 颜色标识：绿色 = 正确，红色 = 错误，黄色 = 部分得分
- 点击某题可展开看完整题干和所有选项

### 7.3 后续操作

- 「重新作答」按钮：清空答案，回到第 1 题
- 「导出成绩」按钮：导出为 PDF 或打印

## 8. 本地存储

### 8.1 存储内容

- `quiz_current`: 当前答题进度（QuizSession 序列化 JSON）
- `quiz_history`: 历史成绩记录（最近 10 条）

### 8.2 行为

- 每次答题自动保存进度到 LocalStorage
- 刷新页面或关闭浏览器后回来，恢复上次进度
- 提交后清除当前进度，追加到历史记录

---

## 9. 边界与限制

- 支持中文题目（UTF-8）
- 单文件最大约 50MB（浏览器内存限制）
- 单次最多解析约 500 题
- 仅支持题号 + 选项 + 答案行的标准格式
- 非标准格式（如分栏排版、图片题、手写批注）可能解析失败

## 10. 不做的事项（YAGNI）

- ❌ 用户登录/注册
- ❌ 题库云端存储/分享
- ❌ AI 辅助解析（第一版不做）
- ❌ 定时/倒计时功能（第一版不做）
- ❌ 题目收藏/错题本（第一版不做）
- ❌ 多文件批量上传（第一版只做单文件）
- ❌ 导出题库
